Affichage Séquentiel (Pro-Tip appliqué)

Pro-Tip : Contrairement à ce qui est suggéré dans l'énoncé, l'initialisation de l'écran lcd_init() doit se faire une seule fois avant la boucle while(1). Si on la met dedans, l'écran redémarre en boucle, ce qui provoque des clignotements désagréables.

// --- AVANT LA BOUCLE WHILE ---
lcd_init(&hi2c1); // Initialisation propre une seule fois

// --- DANS LA BOUCLE WHILE ---
while (1) 
{ 
    // Séquence 1
    lcd_set_cursor(0, 0); 
    lcd_print("Embedded System "); 
    lcd_set_cursor(1, 0); 
    lcd_print("Lab Session     "); 
    HAL_Delay(2000); 
    
    // Nettoyage
    lcd_clear(); 
    
    // Séquence 2
    lcd_set_cursor(0, 0); 
    lcd_print("Day 1           "); 
    lcd_set_cursor(1, 0); 
    lcd_print("Subgroup 1      "); 
    HAL_Delay(1500); 
}

Défi : Affichage dynamique (ADC + I2C)

Réponse à la question 3 du Post-Lab. Utilisation de sprintf() pour formater les données numériques (ADC et Tension) en chaînes de caractères avant de les envoyer sur le bus I2C.

// Bibliothèques requises : #include <stdio.h>
char ligne1[16];
char ligne2[16];

lcd_init(&hi2c1);

while(1) {
    // 1. Lecture de l'ADC
    HAL_ADC_Start(&hadc1);
    HAL_ADC_PollForConversion(&hadc1, 100);
    uint32_t adc_val = HAL_ADC_GetValue(&hadc1);
    HAL_ADC_Stop(&hadc1);

    // 2. Calcul de la tension
    float tension = (adc_val * 3.3f) / 4095.0f;

    // 3. Formatage des chaînes (String Print Format)
    sprintf(ligne1, "ADC: %4lu      ", adc_val);
    sprintf(ligne2, "Tension: %.2fV ", tension);

    // 4. Affichage I2C
    lcd_set_cursor(0, 0);
    lcd_print(ligne1);
    lcd_set_cursor(1, 0);
    lcd_print(ligne2);
    
    HAL_Delay(100); // Rafraîchissement à 10Hz
}

2. Post-Lab : Analyse Technique

Combien de fils économise-t-on en utilisant le module I2C par rapport à un branchement LCD classique (parallèle) ?

Un écran LCD HD44780 classique branché en mode parallèle nécessite une grande quantité de broches du microcontrôleur :

  • Mode 8 bits : 8 broches de données (D0-D7) + 3 broches de contrôle (RS, RW, EN) = 11 broches GPIO.
  • Mode 4 bits (le plus courant) : 4 broches de données (D4-D7) + 3 broches de contrôle = 7 broches GPIO.

En ajoutant le module adaptateur I2C (PCF8574) à l'arrière du LCD, la communication se fait via un bus série synchrone qui ne nécessite que 2 broches (SDA pour les données et SCL pour l'horloge).

Conclusion : On économise entre 5 et 9 broches GPIO du STM32, ce qui est crucial pour libérer des ports pour d'autres capteurs ou actionneurs.

Quelle est l'adresse I2C par défaut de votre écran ? Pourquoi est-elle importante si l'on veut brancher deux écrans sur le même bus ?

L'adresse I2C par défaut d'un module LCD équipé d'une puce PCF8574 est généralement 0x27 (ou 0x3F selon le fabricant de la puce, PCF8574A ou PCF8574T).

Pourquoi est-ce important ? L'I2C est un "Bus". Cela signifie que l'on peut brancher des dizaines de composants différents (capteurs, mémoires, écrans) sur les *mêmes* deux fils (SDA et SCL). Pour que le Maître (le STM32) puisse parler à un composant précis sans que les autres ne répondent, il commence chaque message en envoyant une adresse unique.

Si l'on veut brancher deux écrans LCD sur le même bus I2C, ils ne peuvent pas avoir la même adresse (sinon il y aurait un conflit de données). Il faut donc modifier physiquement l'adresse matérielle du second écran en soudant des ponts (pads A0, A1, A2) situés à l'arrière du module I2C pour lui donner, par exemple, l'adresse 0x26.

Modifiez le code pour afficher la valeur d'un potentiomètre (ADC) en temps réel sur l'écran.

La difficulté principale ici est qu'un écran LCD ne comprend que du texte (des caractères ASCII). Il ne sait pas afficher un nombre entier pur comme 2048 ou un float comme 1.65.

Il faut donc utiliser une fonction de la bibliothèque C standard (<stdio.h>) appelée sprintf(). Elle permet de convertir des variables numériques en chaînes de caractères formatées.

Le code complet a été implémenté et démontré dans l'onglet "Manip 2" du simulateur ci-dessus. Le format %4lu force l'affichage sur 4 chiffres pour éviter que des "fantômes" d'anciens chiffres ne restent à l'écran quand la valeur diminue (ex: passer de 1000 à 999).
